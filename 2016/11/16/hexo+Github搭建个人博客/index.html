hexo+Github搭建个人博客

发表于 2016-11-16

前言:之前用过wordpress来搭建博客，hexo+Github没有实现过，花了点时间来研究一下，虽然过程遇到了一点点小问题，但是最后还是成功了，网上也有很多这些这样的教程，我也只是来记录我捣鼓的过程，我先来写一篇搭建博客的，后面再去拓展主题之类的内容，注意以下内容是在Mac下进行的。

1、准备条件

  • 1、Github帐号 自己去注册，记得邮件验证，不然没法打开你做好的博客
  • 2、Node.js 到官网进行下载安装就行了
  • 3、Hexo 博客系统
  • 4、Git Xcode自带，安装了Xcode就不需要再去安装，这个是用来把本地内容提交到Github的

2、安装Hexo并启动

安装好Git和Node.js之后，新建一个目录，用来存放Hexo的配置文件, 使用终端,然后cd 到创建的目录下使用以下命令进行安装
sudo npm install -g hexo
安装之后使用以下命令进行初始化
hexo init
然后是生成静态页面，使用以下命令进行生成
hexo generate 或者(hexo g)
然后启动服务
hexo server 或者(hexo s)
然后到浏览器的网址输入框输入
http://localhost:4000
就可以看到默认的hexo博客界面

3、Github创建仓库

在Github的登陆界面点击头像的左边的加号图片按钮，然后选择New repository,然后在Repository name的下方的输入框中输入按照以下格式的字符串，格式为yourname.github.io，yourname是你的github的用户名，一定要按照这种格式去做。然后选择public，至于Description填不填都没关系了，Initialize this repository with a README也可以不打勾,然后点击创建按钮就可以进行创建了，然后会显示一个链接，就像https://github.com/xxx/xxx.github.io.git这样的链接，xxx为你的用户名。这是https的链接，如果设置了ssh，也可以使用ssh链接。

4、关联Github仓库

cd 到创建的博客根目录下，找到_config.yml文件，使用以下命令进入进行修改
vim _config.yml
翻到最下面,进行修改，例子如下：

上面的链接其实就是Github上的仓库的链接，修改完之后保存退出，保存退出是按一下esc键，然后按住shift键加冒号键，然后输入wq就可以保存退出了
然后执行以下命令
npm install hexo-deployer-git --save
然后执行以下命令
hexo deploy
可能会出现一个问题就是
fatal: Not a git repository (or any of the parent directories): .git
这个问题的解决方法就是在博客的根目录下对.deploy_git/进行删除然后重新执行hexo deploy,删除这个目录的命令是
rm -rf .deploy_git/
重新hexo deploy之后就可以打开http://xxx.github.io来打开你的博客了，xxx是你的用户名。如果打开网址发现出现404可以稍微等一下，可能提交之后有一点延迟，如果打开成功了，那么至此为止，博客的搭建已经全部完成。
